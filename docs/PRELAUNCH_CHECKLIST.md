# Prelanzamiento B-Side

## Objetivo

Dejar la app lista para iterar tranquilo, validar producto y gusto personal
antes de pensar en produccion.

## Prioridad alta

1. Login real con email y proveedor social.
2. Backend para perfiles, reseñas, listas, likes, comentarios y mensajes.
3. Terminos y condiciones.
4. Politica de privacidad.
5. Lineamientos de comunidad y moderacion.
6. Reportes, bloqueo y herramientas basicas de seguridad.
7. Configurar buckets para avatar y wallpaper.

## Prioridad media

1. Notificaciones push.
2. Seguidores, seguidos y feed personalizado.
3. Busqueda enriquecida con mas metadata.
4. Compartir contenido con deep links.
5. Editor de perfil mas completo.

## Monetizacion para evaluar mas adelante

1. Freemium con perfiles Plus.
2. Listas premium, stats avanzadas y top historicos.
3. Afiliacion a vinilos, merch o tickets.
4. Promociones de artistas o lanzamientos.
5. Patrocinios o espacios curados.

## Seguridad y legal

- Definir responsable de datos y canal de contacto.
- Aclarar que contenido es del usuario y que metadata musical es de terceros.
- Establecer politica de bajas y reportes.
- Definir edad minima de uso.
- Revisar almacenamiento de datos y backups.
- Incluir terminos de suscripcion si existe Plus pago.

## Senales de que la app ya esta publicable

- El onboarding y login funcionan.
- Hay backend estable.
- Existe recuperacion de cuenta.
- Las politicas estan accesibles dentro de la app.
- Existe moderacion minima.
- Hay metricas basicas para entender uso y errores.
